# Source-of-Record Separation

| Domain | Source of record | CPCF role |
|---|---|---|
| Mission, task, lease, workcell, residual workflow | CCR | inspect, project, route |
| Finite checks, plans, BIT, certificates | PIC | invoke, capture, project |
| Verifier ecology | VEK | project, handoff |
| Claims, support, obligations | FOST | project, handoff |
| Observable promotion | CIMT | project, handoff |
| Capability capital | ALT | candidate handoff, decision capture |
| Proof-carrying skills | PCS | candidate handoff, receipt capture |
| Governed memory | CMGL | candidate handoff, decision capture |
| Problem frames | PFG | candidate handoff |
| Scoped authority | no-meta-authority-runtime | request handoff |
| Future-claim authority | DFCC | candidate handoff |
| CAS, projections, closure, seeds, barriers, planner | CPCF | authoritative |

External reports remain immutable. CPCF stores raw bytes and projections. Its SQLite file indexes
only CPCF CAS objects and can be rebuilt.
