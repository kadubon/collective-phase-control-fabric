# Quarantined Legacy Adapter Targets

CCR 1.6.0 and PIC 1.1.0 are read-only compatibility targets last locally rechecked on 2026-07-13.
A version handshake precedes every legacy inspection operation. Editable package metadata is
provenance only and is not treated as the source interface version.

No mapping is enabled for v0.6 promotion. Enabling one requires a v0.6 typed capability, independent
approval, digest-scoped runner materials, a closed output model, and exact pointer reconstruction.
Reconnaissance is not authority to execute either upstream system.

Legacy CCR operation: `agent_explain`. Legacy PIC operations: `agent_explain`, `agent_check`,
`phase_plan`, and `canonical_readiness`. PIC probes disable live connectors. Safe commands returned
by either upstream are never executed.

Durable legacy bytes can enter only through copy-on-write legacy import and remain quarantined.
Native projections bind exact raw digests and JSON pointers to registered v0.6 models. Changed
shapes fail closed.

CCR `storage_doctor` is deliberately not registered because prior audit observed that it can
materialize `ccr.sqlite` in a selected root. Its report schema remains available only for read-only
compatibility analysis.
